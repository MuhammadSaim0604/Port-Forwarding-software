Documentation

Learn how to use the Tunnel Management System

Getting Started

The Tunnel Management System allows you to expose local services to the internet securely through encrypted tunnels.

  • Create a tunnel with a name and local port
  • Verify your tunnel through the verification link
  • Download and run the client to establish connection
  • Access your service via the public address

Creating a Tunnel

To create a new tunnel:

  1. Navigate to the Dashboard tab
  2. Fill in the tunnel name (e.g., "My Web Server")
  3. Specify the local port your service runs on
  4. Select the protocol (TCP for web, UDP for gaming)
  5. Click "Create Tunnel"

Verifying Tunnels

After creating a tunnel, you must verify it before use. Click the verification link shown in the tunnel card or in the alert message. This security step ensures only authorized users can create tunnels.

Running the Client

Download the client file (.bat for Windows) from the tunnel card and run it. The client will:

  • Install required Python dependencies
  • Connect to the server using WebSocket
  • Forward traffic between public port and local service
# Example: Running the client manually python tunnel_client.py https://your-server.com TOKEN_HERE TUNNEL_ID LOCAL_PORT

Protocol Support

The system supports multiple protocols:

  • TCP: Best for HTTP, HTTPS, SSH, databases, and most web services
  • UDP: Ideal for gaming, VoIP, DNS, and streaming applications
  • BOTH: Forwards both TCP and UDP traffic on the same port

Common Use Cases

  • Web Development: Share your local development server (port 3000, 8080, etc.)
  • Game Servers: Host Minecraft, Counter-Strike servers using UDP
  • API Testing: Expose your local API for testing with external services
  • IoT Devices: Connect to devices behind NAT/firewall
  • Remote Access: SSH into your local machine from anywhere

Troubleshooting

Tunnel shows as inactive:

  • Ensure the client is running
  • Check that the tunnel is verified
  • Verify the local service is running on the specified port

Cannot connect to public address:

  • Confirm tunnel status is "Active"
  • Check firewall settings on your local machine
  • Ensure the local port matches your service

API Endpoints

GET /api/tunnels - List all tunnels POST /api/tunnels - Create new tunnel DELETE /api/tunnels/:id - Delete tunnel GET /verify/:code - Verify tunnel GET /download/:id - Download client
